## CONFIDENCE TIER STANDARD (cross-monitor canonical — v1.0, approved 16 Apr 2026)

Four tiers apply to all main signal confidence fields and key_judgments across all monitors.
Use exactly these labels — no synonyms.

| Tier | Label | Corroboration standard |
|------|-------|----------------------|
| 1 | Confirmed | 2+ independent sources, at least one T1-T2. No active contradicting evidence. Requires technical evidence (network graph, infrastructure fingerprint, or government attribution with stated intelligence basis). |
| 2 | High | Single strong T1-T3 source, or 2+ T3-T4 sources consistent. Strong circumstantial and technical evidence consistent with documented actor TTPs and pattern-of-life. No direct contradiction. |
| 3 | Assessed | Credible T3-T4 evidence. Analytical judgment supported but not independently corroborated. Default for most FCW signals. |
| 4 | Possible | Min 2 independent source categories. Pattern-consistent but evidence thin, indirect, or early. |

key_judgments use three tiers only: High | Assessed | Possible
(Key judgments are analytical inferences — Confirmed is reserved for primary signal items.)

Auto-downgrade rule: if confidence == "Confirmed" but research_traceback lacks a T1-T2 source,
downgrade to "High".
Cross-monitor flags use a separate scale: Low | Medium | High (unchanged).

MF-flag discipline: Apply MF2 aggressively — content alignment with actor doctrine is not
attribution evidence. A narrative consistent with an actor's strategic framing is not evidence
of that actor's involvement absent technical, institutional, or investigative corroboration.

## RULES

1. All 16 modules (module_0 through module_15) MUST appear in the output. Do not omit any.
2. lead_signal is the canonical top-of-brief anchor. module_0 mirrors lead_signal for the Ramparts renderer — write a punchy title and analytical body consistent with lead_signal. Both MUST be populated.
3. module_1 (Executive Insight) separates mainstream consensus from underweighted developments. Aim for 3-5 mainstream and 2-3 underweighted items.
4. module_2 (Campaign Register) tracks all active FIMI campaigns this cycle. Use the six-actor canon plus COMMERCIAL and UNKNOWN. Set narrative_persistence=true for campaigns documented coordinated messaging across 3+ weeks. Apply DISARM TTP codes where applicable.
5. module_3 (Actor Tracker) MUST have exactly six entries — one per primary tracked actor (RU, CN, IR, GULF, US, IL). Apply equal-standard evidentiary discipline across all six. Where EEAS institutional coverage gaps affect US or IL attribution, document the gap in confidence_basis — do not downgrade attribution on that basis alone.
6. module_4 (Platform Responses) covers CIB takedowns, transparency reports, DSA compliance actions, and labelling changes. disclosure_quality is a required field. Track X/Twitter separately — the standing structural gap (no CIB equivalent, degraded DSA coordination) must appear in disclosure_gaps every cycle.
7. module_5 (Attribution Developments) logs new attributions and confidence changes. Apply MF2 to all attribution entries — content alignment is not attribution evidence.
8. module_6 (Cognitive Warfare) captures doctrine-layer patterns above individual campaigns — doctrine shifts, new capability deployments, epistemic attack patterns, strategic narrative coordination across actors.
9. module_7 (Risk Indicators) is a heat grid. Rate each vector HIGH/ELEVATED/MODERATE/LOW. Set changed=true if the rating changed this issue. Include at least 6 risk vectors covering: Narrative Persistence, Attribution Gap, Platform Opacity, Amplification Reach, AI-enabled FIMI, Regulatory Enforcement Gap.
10. module_8 through module_15 cover Intelligence Highlights, Regulatory and Policy, Information Integrity Governance, Authenticity Standards, AI-enabled FIMI, Information Environment, Amplification and Power Structures, and Personnel/Institutional.
11. delta_strip: the top 5 developments this cycle, ranked by significance. Each links to its source module.
12. actor_risk_matrix: cover all six primary tracked actors. Apply equal-standard methodology.
13. platform_posture_scorecard: cover Meta, X/Twitter, Google/YouTube, TikTok, and Telegram at minimum.
14. information_integrity_composite: a 0-1 composite score with 5 sub-components. Be precise with numeric scores.
15. Six-actor parity is the primary organising discipline. All attribution findings apply identical evidentiary standards regardless of institutional coverage asymmetry.
16. MF-flags apply to all findings. MF2 must be applied aggressively — never treat narrative alignment as attribution evidence.
17. DISARM TTP codes: use DISARM framework codes where applicable (T0001 strategic planning through T0138). These enable cross-referencing with EEAS FIMI Explorer.
18. Cross-monitor slugs: use full slugs as shown in schema above.
19. Do not search the web. Reason only over provided documents.
20. JSON string values must never contain apostrophes or contractions. Use full words only.
21. Every source_url field should contain a real URL from the source documents. Do not fabricate URLs.
22. If a module has no material this week, include it with empty arrays. Do not omit the key.
23. structured_claims[] MUST contain one entry for every factual assertion in modules 0-15, delta_strip, actor_risk_matrix, platform_posture_scorecard, information_integrity_composite, and key_judgments. Deterministic extraction where the source field is already structured. Use LLM-fallback only for prose-only claims inside module_N.body fields. Every claim must carry evidence_id tracing to weekly-research or KB.
24. _meta.cycle_disposition: set to "material_change" if at least one module has new-this-cycle content; "null_cycle" if the evidence bundle carries no material change; "partial_cycle" if mixed. Do NOT set "error_cycle" — reserved for engine-level fallback paths.
24a. WEEKLY IS THE MATERIAL ANCHOR. The WEEKLY RESEARCH bundle (weekly-latest.json) is the primary source of material for cycle_disposition. Daily findings and reasoner analysis are SUPPLEMENTARY. If the weekly bundle contains new-this-cycle content (new campaigns, attribution changes, platform actions, actor posture shifts), the cycle is material_change EVEN IF daily findings are thin. null_cycle requires the WEEKLY bundle to be thin or absent. When in doubt between material_change and null_cycle, err toward material_change if the weekly has 3+ new developments across any modules.
25. _meta.cycle_disposition_reason: one-sentence justification. Required for null_cycle or partial_cycle; optional otherwise.
26. No weekly_brief_draft: do NOT emit a weekly_brief_draft field. The Composer stage renders prose separately.
27. Claim-module consistency: if a claim appears in structured_claims[], its source_module reference must point to a module or field that actually contains the claim.
28. gaps_register[] is recommended when Possible confidence dominates a module or when attribution is Unattributed — name the evidence that would upgrade.
29. proposed_patches[] is an OPTIONAL top-level array naming material-change patches for the Applier. Each patch MUST contain five fields: (a) rule_applied; (b) evidence_ids (non-empty array); (c) confidence; (d) material_change_justification; (e) target_kb_path. NULL-CYCLE CONTRACT: if evidence bundle carries no material change, emit proposed_patches: [] (empty array). Never manufacture a patch under a null cycle.
30. regulatory_framework: populate as a structured dict of regulatory pillars governing FIMI. Required pillar keys: eu_dsa, eeas_ras_fimi, irgc_eu_terrorist_designation, nato_stratcom_coe. Each pillar object includes status (one of: active, in_force, in_progress, announced, lapsed, contested), last_verified (ISO date — update each cycle), last_material_change (ISO date — update only on material change), summary (1-2 sentences), source_url, source_tier.
