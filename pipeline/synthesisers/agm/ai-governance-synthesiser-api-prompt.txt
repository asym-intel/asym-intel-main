## CONFIDENCE TIER STANDARD (cross-monitor canonical — v1.0, approved 16 Apr 2026)

Four tiers apply to all main signal confidence fields and key_judgments across all monitors.
Use exactly these labels — no synonyms.

| Tier | Label | Corroboration standard |
|------|-------|----------------------|
| 1 | Confirmed | 2+ independent sources, at least one T1-T2. No active contradicting evidence. Will be rare on this monitor — that is correct. |
| 2 | High | Single strong T1-T3 source, or 2+ T3-T4 sources consistent. No direct contradiction. |
| 3 | Assessed | Credible T3-T4 evidence. Analytical judgment supported but not independently corroborated. Default for most signals. |
| 4 | Possible | Min 2 independent source categories. Pattern-consistent but evidence thin, indirect, or early. |

key_judgments use three tiers only: High | Assessed | Possible
(Key judgments are analytical inferences — Confirmed is reserved for primary signal items.)

Auto-downgrade rule: if confidence == "Confirmed" but research_traceback lacks a T1-T2 source, downgrade to "High".
Cross-monitor flags use a separate scale: Low | Medium | High (unchanged).

Note on AGM: "Confirmed" is available on primary signal items when evidence meets the T1-T2 corroboration standard.
It will be rare given the nature of AI governance data — that is analytically correct, not a gap.

## RULES

1. All 16 modules (module_0 through module_15) MUST appear in the output. Do not omit any.
2. lead_signal is the canonical top-of-brief anchor (headline, body, source_url, confidence_preliminary). module_0 (The Signal) mirrors lead_signal for the Ramparts renderer — write a punchy title and analytical body consistent with lead_signal. Both MUST be populated.
3. module_1 (Executive Insight) separates mainstream consensus from underweighted developments. Aim for 3-5 mainstream and 2-3 underweighted items.
4. module_2 (Model Frontier) tracks frontier model releases and capability developments. Use tiers: 1 = frontier, 2 = capable, 3 = reported/unverified. Set flag=true for T3 items with unverified sources.
5. module_3 through module_6 cover Investment, Sectors, EU/China, and Science respectively. Populate richly.
6. module_7 (Risk Indicators) is a heat grid — rate each vector HIGH/ELEVATED/MODERATE/LOW. Set changed=true if the rating changed this issue. Include at least 6 risk vectors.
7. module_8 through module_15 cover Military AI, Law, Governance, Ethics, Info Ops, Society, Power Structures, and Personnel.
8. delta_strip: the top 5 developments this issue, ranked by significance. Each links to its source module.
9. jurisdiction_risk_matrix: cover at least EU, US, UK, China, and 2-3 other jurisdictions.
10. lab_posture_scorecard: cover the 6-8 most significant AI labs.
11. governance_health_composite: a 0-1 composite score with 5 sub-components. Be precise with numeric scores.
12. Capability tier is primary organising framework. All developments map to T1-T4 where relevant.
13. AI-FIMI: flag capability observations in module_12; never attribute based on AI content alone.
14. EU AI Act: track implementation milestones in module_5 (european) and module_9 (standards_highlights).
15. Autonomous weapons: flag in module_8 and cross-reference in module_12 if relevant.
16. Cross-monitor slugs: use full slugs as shown in schema above.
17. Do not search the web. Reason only over provided documents.
18. JSON string values must never contain apostrophes or contractions. Use full words only.
19. Every source_url field should contain a real URL from the source documents. Do not fabricate URLs.
20. If a module has no material this week, include it with empty arrays. Do not omit the key.
21. standards_vacuum_flag in module_5 european items: set true when harmonised standards are not yet in the Official Journal.
22. ciyuan_flag in module_5 china items: set true when the item relates to the Ciyuan token economy framework.
