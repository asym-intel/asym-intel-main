## RULES

1. Stress regime is conjunctive — requires evidence across two+ indicator domains.
2. `system_average` is the arithmetic mean of all indicator domain stress scores: Green=0, Amber=-0.33, Red=-0.66, Crisis=-1.0. Range: 0.0 (all green) to -1.0 (all crisis). Negative = stress. CRITICAL threshold: below -0.50.
3. `conviction` reflects how many domains corroborate the regime: 1 domain = LOW, 2-3 = MEDIUM, 4-5 = HIGH, 6 = VERY HIGH.
4. `lead_signal` is REQUIRED. The headline must be factual and specific — not a regime label. "Liberation Day tariff package activates 10-point baseline surcharge" not "Macro stress elevated".
5. `tail_risks`: produce 5-8 risks. `likelihood` and `impact` are numeric 0.0-1.0. Likelihood sourced from Horizon Matrix probabilities. Impact = (affected asset classes / 8) × avg score change × 1.5. `label` is a short name. `note` describes the risk including trigger condition.
6. `confidence_preliminary` is advisory only — Publisher strips the _preliminary suffix for the final report.
7. Tariff tracker: document all new measures and retaliation chains this week.
8. Central bank tracker: note divergence between major institutions.
9. Null signal: set true only if all six domains Green/stable, no regime delta.
10. Cross-monitor slugs: use full slugs as shown in schema above.
11. Do not search the web. Reason only over provided documents.
12. JSON string values must never contain apostrophes or contractions. Use full words only (e.g. "this weeks" not "this week's", "does not" not "doesn't"). Apostrophes break JSON parsing.
13. All fields in the schema are REQUIRED. Do not omit stress_regime_preliminary.regime, stress_regime_preliminary.conviction, stress_regime_preliminary.system_average, or lead_signal.
