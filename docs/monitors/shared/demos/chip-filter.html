PI-XX — chip-filter

Universal chip / chip-filter primitive. Replaces every bespoke ov-chip / risk-chip class in the monitor commons.

Authority: P5 (universal affordance), P6 (canonical class set, no bespoke duplicates), P12 (product not library — demo proves the primitive is product, not shelfware), P13 (universal primitive — shared across commons §1 and commercial §2 consumers), §Zero-leftover discipline (no bespoke class CSS remains in base.css after migration).

Consumers: Commons §1 — overview pages on every monitor (ai-governance, european-strategic-autonomy, fimi-cognitive-warfare, macro-monitor, conflict-escalation, democratic-integrity, environmental-risks), plus future filter-bar consumers on dashboard pages. Commercial §2 — Advennt jurisdiction pages (Wave-3, pending).

1 · Default state — display-only chips

Representative real-world usage. Overview pages emit a chip row of metadata pills — cadence, scope, tracking dimensions. These are display-only (no filter behaviour); semantic markup uses <span class="chip"> rather than <button>.

Weekly · governance watch 16 modules scored Regulatory + capability tracking

2 · Edge state — interactive filter + long label + selected

Boundary conditions. Interactive filter chips use <button> with aria-pressed. Selected state applies the --chip-bg-selected token set. The "long label" chip demonstrates that chips wrap to the next row rather than overflowing the container.

3 · Empty / null state — graceful fallback

Honesty contract (PI-1, graceful-fallback). When the upstream filter source returns zero options the primitive emits .chip-filter__empty rather than an empty container. This is the canonical "no data" affordance for the primitive; consumers do not need to render any chip elements.
