PI-XX — freshness-badge

Universal freshness affordance. Communicates how recently a data surface was updated, with state variants and a strict empty-state contract.

Authority: P3 (honest absence over fabricated precision), P5 (universal affordance), P12 (product not library — demo proves the primitive is product, not shelfware), P13 (universal primitive — shared across commons §1 and commercial §2 consumers). Empty state honours PI-1 strict (schema-version graceful-fallback): when data-updated is absent or empty the element is omitted via CSS, never rendered as broken markup.

Consumers: Commons §1 — every monitor overview / dashboard / report page surfaces a freshness affordance. Commercial §2 — Advennt jurisdiction pages (Wave-3, pending).

3 · Empty / null state — graceful fallback (PI-1 strict)

Honesty contract. Two empty variants render as nothing via display: none — there is no "—" placeholder, no broken markup, no misleading zero. The primitive is intentionally invisible when data is absent.

(invisible) data-updated="" → display: none
(invisible) data-updated attribute absent → display: none

Neither variant occupies layout space. This satisfies PI-1 strict: absent _meta.last_updated in the upstream schema → element omitted, never broken markup.
