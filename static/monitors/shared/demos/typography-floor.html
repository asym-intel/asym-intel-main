PI-XX — typography-floor

Universal type-scale floor and body line-height enforcement. Demo page proves the primitive is product, not shelfware.

Authority: P12 (product not library — demo proves the primitive is product, not shelfware), P13 (universal primitive — this token set is shared across commons §1 and commercial §2 consumers), §13-T1 (typography-floor enforcement; --text-min is the absolute floor for any rendered text in monitor UI).

Consumers: Commons §1 — all monitors under static/monitors/. Commercial §2 — Advennt emit (Wave-3, pending). Both surfaces inherit the floor through base.css.

1 · Default state — body text at --text-base

Representative real-world usage. Body copy across monitor pages renders at --text-base with the body line-height token applied via :root + body selectors in base.css.

The typography floor sets a single absolute minimum (--text-min) below which no rendered text in the monitor UI is permitted. Body copy sits at --text-base with body line-height inherited from :root. This block uses font-size: var(--text-base) verbatim — no inline overrides, no shadowed tokens.

2 · Edge state — text at the --text-min floor

Boundary condition. This block is sized at --text-min — the absolute floor. No text in monitor UI may be smaller than this. Visual delta vs example 1 should be perceptible but legible at AA contrast.

Tiny chrome captions, metadata strips, and table footnotes that previously sized below the floor have been pinned to --text-min. Anything below is a violation and will fail the typography-floor lint sweep.

3 · Empty / null state — graceful fallback

Honesty contract (PI-1, schema-version graceful-fallback). Typography-floor is a token primitive: it applies whether or not data is present. The "empty" state of this primitive is the absence of a paragraph — and the absence still respects line-height tokens on neighbouring elements. No broken markup, no shifted layout.

  • --text-min — absolute floor (also aliased --text-xs)
