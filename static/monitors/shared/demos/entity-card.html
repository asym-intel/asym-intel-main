L2 — entity-card

Shared-layer card primitive. Hosts headline + optional subhead + meta (freshness-badge) + optional CTA. The single visual contract for jurisdiction cards, signal cards, monitor cards, and archive items across the fleet.

Authority: P1 (schema-driven projection), P5 (universal composition primitive), P12 (product not library), P13 (no per-page edits — adding a new card is a data render, not an HTML touch). Empty state honours PI-1 strict: when headline is absent the card is omitted entirely; when freshness.updatedAt is absent the freshness-badge is omitted but the rest of the card still renders.

1 · Default composition — headline + freshness-badge

Representative usage on a listing page. entity-card.js consumes a JSON descriptor and emits a card hosting the freshness-badge primitive (BRIEF 04 contract).

2 · Typography floor — three font-size sample points

All text inside an entity-card respects base.css v1.8 line-height + body defaults. Below: the same card rendered at --text-sm, --text-md, and --text-lg headline sizes — line-height stays ≥ 1.3 (P12 sub-rule 2) and body remains ≥ 16px (P12 sub-rule 1).

3 · PI-1 graceful-fallback

Two descriptors are passed below: one with a missing headline (returns null from the builder — card omitted entirely), one with no freshness data (card renders, badge omitted via PI-1 CSS rule in freshness-badge.css).

// passed to AsymEntityCard.renderInto:
[
  { headline: '' },                       // → null, no card
  { headline: 'Renders without freshness',
    subhead: 'No updatedAt provided.',
    eyebrow: 'PI-1 fallback' },
  { headline: 'Renders with freshness',
    subhead: 'updatedAt: today',
    eyebrow: 'PI-1 happy path',
    freshness: { updatedAt: '' } }
]

4 · Pending + CTA variants

A pending card is dimmed and non-interactive even if an href is supplied. A CTA renders as either an inline-flex anchor (when the host is a <div>) or as a label (when the host is itself an <a>).
